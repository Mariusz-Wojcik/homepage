TYDZIEŃ III

Znaczników jest więcej...

  • Prezentują się one następująco:
    • <br> - nowa linia (znacznik pojedynczy)
    • <b> - pogrubienie oraz <i> - kursywa/italic - Nie mają dużego znaczenia semantycznego - służą do wyróżnienia fragmentu z tekstu otaczającego
    • <strong> - pogrubienie - podkreśla wysoką istotność i poważność frazy w nim zawartej
    • <em> - kursywa/italic - podkreśla, że tekst może mieć inne znaczenie (np. ironia)
    • <element id="moj-identyfikator"> - atrybut id identyfikuje element - przydaje się do wewnętrznej nawigacji
    • <a> - linki / łącza wewnątrz dokumentu
      <a href="" target=""> opis łącza </a> - łącze zewnętrzne
      <a href="#atrybut_id"> opis łącza </a> - łącze wewnętrzne
    • <nav> - nawigacja
    • <img src="" alt=""> - obrazek o podanym adresie - a jak nie dodasz alt="" to Ci walidator przypomni ;)
      <img> jest również samozamykający jak <br>
      alt - tekst alternatywny dla przeglądarki żeby było wiadomo co jest na obrazku

Trochę o nawigacji...

  • Przeglądarka analizuje tylko to co jest w <main>.
  • Każdą treść która jest powtarzalna wrzucamy poza <main> na przykład w <header> który na każdej stronie będzie wyglądał tak samo.
    <nav> wrzucamy właśnie w <header> poza <main>.

TYDZIEŃ II

Podstawy HTML

  • HTML = HyperText Markup Language - odpowiada za treść a dokładniej za jej właściwe opakowanie. Nie jest to język programowania tylko język znaczników.
  • Semantyczny = mający znaczenie
  • HTML składa się ze znaczników/tagów/elementów
    <znacznik atrybut=”wartość”> treść </znacznik>
  • Poznane znaczniki:
    • <main> - Główna część dokumentu
    • <section> - Sekcje dokumentu
    • <article> - Samodzielne treści (np. Gazeta, dział sportowy -czyli sekcja- i tu wjeżdża na białym koniu samodzielny artykuł)
    • <header> - Nagłówek - dokumentu / sekcji / artykułu (Każda sekcja i każdy artykuł posiada nagłówek, możesz być pewien)
    • <footer> - Stopka - dokumentu / sekcji
    • <h1-h6> - Tekstowe nagłówki tytułowe - dokumentu /sekcji
    • <p> - Paragraf / akapit
  • Jak już myślisz, że stworzyłeś kawał niezłego kodu to w pierwszej kolejności zaprasza walidator HTML
